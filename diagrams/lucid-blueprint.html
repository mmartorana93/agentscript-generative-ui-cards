5 · How to rebuild this in Lucid

  1. Don't start from a blank canvas. Reference Architecture Gallery → architect.salesforce.com/diagrams; pick a System Landscape / Solution Architecture template as the base.
  2. Load shapes. Shapes panel → Standard Libraries → Salesforce Architecture Diagrams → Use Selected Shapes. Import React / Slack / Tableau assets as custom shapes (see the asset table above for sources).
  3. Lay out the three diagrams using this HTML as the pixel reference: box positions, arrows, labels, colors.
  4. Drop one official icon per ◇ ICON SLOT. Snap SF icons into the card headers; place React/Slack/Tableau where noted.
  5. Keep the semantic colors (agentic #FDE9D9/#F4543A, deterministic #DDF3E4/#26B864, UI #E3F1FB/#83D9F2) so Lucid matches the repo's Mermaid.
  6. Export PNG (and/or SVG) → commit under diagrams/ and reference from the READMEs with ![](diagrams/…png). Keep the Mermaid blocks as a text fallback.
Reality check on the Lucid MCP (mcp.lucid.app/mcp): it can create the diagram structure (nodes, arrows, layout, colors, text) from a prompt and export PNG — great for scaffolding these three layouts fast. It does not reliably drag the official Salesforce shape-library icons into nodes; that step stays in the Lucid editor. So: MCP for the skeleton, editor for the official icons.
